---
title: Disputes
description: Create and manage disputes for card transactions.
---

Disputes allow you to challenge card transactions on behalf of your customers. Each dispute tracks its lifecycle from filing through resolution.

---

## Create Dispute

```http
POST /disputes
```

### Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `reason` | string | ✓ | `fraudulent`, `duplicate`, `value-not-received`, `general` |
| `source` | string | ✓ | Authorization code of the disputed transaction |
| `text` | string | | Detailed description of the dispute |
| `documents` | array | | Array of evidence document URLs |
| `user` | string | | User code |

### Example Request

```json
{
  "reason": "fraudulent",
  "source": "auth_xxxxx",
  "text": "Customer did not authorize this transaction",
  "documents": [
    "https://example.com/evidence/screenshot.png"
  ]
}
```

## List Disputes

```http
GET /disputes
```

Standard [pagination, filtering, and sorting](/getting-started/#pagination) parameters apply.

### Download as CSV

```http
GET /disputes/csv
```

---

## Get Dispute

```http
GET /disputes/:id
```

Returns a single dispute by ID.

---

## Update Dispute

```http
PUT /disputes/:id
```

Update dispute details — add more evidence or update the description.

| Field | Type | Description |
|---|---|---|
| `text` | string | Updated dispute description |
| `documents` | array | Updated evidence URLs |
| `user` | string | User code |

---

## Close Dispute

```http
PUT /disputes/:id/close
```

Close a dispute. Closed disputes cannot be reopened.
