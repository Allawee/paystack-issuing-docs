---
title: Events
description: Query and retry webhook events.
---

Events represent webhook delivery records. Each event corresponds to a notification sent (or attempted) to your webhook endpoints.

---

## List Events

```http
GET /events
```

Standard [pagination, filtering, and sorting](/getting-started/#pagination) parameters apply.

---

## Get Event

```http
GET /events/:id
```

Returns a single event by ID, including delivery status and payload.

---

## Retry Event

```http
POST /events/:id/retry
```

Manually retry delivery of a failed event to its webhook endpoint.
