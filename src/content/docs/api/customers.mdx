title: Customers
description: Create and manage customers — individuals and businesses.
---

Customers represent the end-users of your financial products. Allawee supports both individual and corporate customer types with tiered KYC verification.

---

## Create Customer

```http
POST /customers
```

### Individual Customer

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | ✓ | Customer display name |
| `reference` | string | ✓ | Your unique reference |
| `type` | string | ✓ | `individual` |
| `claims.individualInformation` | object | | Personal details |
| `claims.individualAddress` | object | | Address information |
| `claims.individualIdentity` | object | | Identity document |
| `verifications` | array | | KYC tier verifications |
| `metadata` | object | | Custom key-value data |

#### Individual Information

| Field | Type | Description |
|---|---|---|
| `title` | string | Mr, Mrs, etc. |
| `gender` | string | Gender |
| `nationalityCode` | string | Country code |
| `firstName` | string | First name |
| `middleName` | string | Middle name |
| `lastName` | string | Last name |
| `phoneNumber` | string | Phone number |
| `dateOfBirth` | string | Date of birth |
| `email` | string | Email address |

#### Individual Identity

| Field | Type | Description |
|---|---|---|
| `type` | string | `bvn` |
| `id` | string | Identity number |
| `url` | string | Document URL |
| `issuingCountry` | string | Issuing country code |

#### Verification Tiers

| Tier | Description |
|---|---|
| `tier-1` | Basic verification |
| `tier-2` | Standard verification |
| `tier-3` | Enhanced verification |

### Example Request (Individual)

```json
{
  "name": "John Doe",
  "reference": "cus-ref-001",
  "type": "individual",
  "claims": {
    "individualInformation": {
      "firstName": "John",
      "lastName": "Doe",
      "phoneNumber": "+2348012345678",
      "dateOfBirth": "1990-01-15",
      "email": "john@example.com"
    },
    "individualAddress": {
      "city": "Lagos",
      "state": "Lagos",
      "countryCode": "NG",
      "addressLineOne": "123 Main Street"
    },
    "individualIdentity": {
      "type": "bvn",
      "id": "12345678901"
    }
  },
  "verifications": ["tier-1"]
}
```

### Corporate Customer

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | ✓ | Business name |
| `reference` | string | ✓ | Your unique reference |
| `type` | string | ✓ | `corporate` |
| `claims.businessInformation` | object | | Business details |
| `claims.businessIdentity` | object | | Business registration (type: `cac`) |
| `claims.businessDirectors` | array | | Company directors |
| `claims.businessAddress` | object | | Business address |

## List Customers

```http
GET /customers
```

Standard [pagination, filtering, and sorting](/getting-started/#pagination) parameters apply.

---

## Get Customer

```http
GET /customers/:id
```

Returns a single customer by ID.

---

## Update Customer

```http
PUT /customers/:id
```

Update customer information. Accepts the same fields as creation.
