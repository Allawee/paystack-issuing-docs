---
title: Payment Requests
description: Create virtual accounts for receiving specific payment amounts.
---

Payment requests create dedicated virtual accounts for collecting specific amounts. They are single-use — once the correct amount is received, the request is fulfilled. If an incorrect amount is received, it is automatically refunded.

---

## Create Payment Request

```http
POST /payment-requests
```

### Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `method` | string | ✓ | `bank-transfer`, `transfer`, `fx-transfer`, `fx-charge`, `billing`, `local-transfer` |
| `options` | object | ✓ | Request options |
| `currency` | string | ✓ | Currency |
| `amount` | number | ✓ | Expected amount in minor units (kobo/cents) |
| `customer` | string | | Customer ID |
| `reference` | string | | Your unique reference |
| `account` | string | | Settlement account |
| `metadata` | object | | Custom key-value data |

### Options

| Field | Type | Description |
|---|---|---|
| `name` | string | Name for the payment request account |
| `bank` | string | Preferred bank (optional) |

### Example Request

```json
{
  "method": "bank-transfer",
  "options": {
    "name": "Invoice #1234"
  },
  "currency": "NGN",
  "amount": 1500000,
  "customer": "cus_xxxxx",
  "account": "act_xxxxx",
  "reference": "inv-1234"
}
```

## List Payment Requests

```http
GET /payment-requests
```

Standard [pagination, filtering, and sorting](/getting-started/#pagination) parameters apply.

---

## Get Payment Request

```http
GET /payment-requests/:id
```

Returns a single payment request by ID.
